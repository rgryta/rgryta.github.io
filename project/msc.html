Mobile device user modelling based on sensoric data from the device

Master of Science in Engineering thesis project.
As part of the thesis, UserModeller platform for researching mobile device user models was developed. User behavior models were built based on data collected from device's sensors.
Basic components of the UserModeller system are: an application for measurements collection, a database, an application used for managing communication within the system, and a library of machine learning methods used for creating models. The paper presents the results of experiments, which aimed to compare models created using various algorithms from the library.
UserModeller system contains four components.

DataProvider

Android application that provides smartphone user with a number of tests imitating screen unlocking mechanism. This application was also released within Play Store in beta program for easier installation.
Java SQLite Android
GitHub Page Play Store (deprecated)

DataCollector

Azure SQL Server service that stores all the data from all internet-enabled smartphones using the provided application. Integration between the cloud-enabled database and clients was managed by Azure Functions solution - idFunction.
Java JDBC Azure Functions Azure SQL Server
GitHub Project

DataUnifier

Application responsible for dumping all the data from cloud SQL database and storing it locally. Then parses the data creating all users' characteristics storing them within csv files for later analysis.
C++ JNI CMAKE
GitHub Project

SolverLibrary

Library of defined tests and algorithm implementations comparing different modelling approaches for user identification and unauthorised access detection.
